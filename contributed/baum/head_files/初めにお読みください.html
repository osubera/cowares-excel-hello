このソフトにできること

ファイルの先頭を見て、テキストを判定する。

エクセルシート上の、ファイルリストを読み、各ファイルの先頭を軽く読んで、テキストかバイナリかを判定する。 可能ならテキストの文字コード判定なども行う。

ご使用の手引き - とりあえず使ってみる

エクセル2007 以降はアドインリボンで操作
エクセル2003 以前はツールバーで操作
  1. このソフトウェアを開きます。
  2. ファイルリストのあるセル範囲を選択します。
  3. ツールバーの「判定実行」ボタンを押します。
  4. ファイルリストの右方に、判定結果等が書き込まれました。

ご使用の手引き - 詳しく知りたい

  1. このソフトは判定結果をファイルリストのあるシートに書き込みます。
    • 判定結果が、ワークシートの既存データを上書きする可能性があります。
    • ファイルリストを含む連続テーブルの右端に、結果を書き込みます。
    • 連続せず、飛び飛びに書き込みがある場合、この判定がうまく働かず、上書きが起こります。
  2. ファイルリストはタイトルを持ち、縦一列に並ぶものに限ります。
    • ファイルのフルパスを並べる必要があります。
    • タイトル行が無くても動作しますが、結果の書式が崩れます。
    • 行によって異なる列を選んでも動作しますが、結果は行ごとに同じ位置に書き出すため、同一行に２つ以上あれば上書きが起こります。
    • 「ファイル一覧表 list_files」が出力した B 列を、そのまま使うことができます。
  3. 可視セルのみを対象とするので、オートフィルタなどで対象を絞り込んで実行できます。
  4. 複数セルが選ばれている場合は、それが対象となり、単一セルが選ばれている場合は、そこを始点として下方向に連続した範囲を対象とします。
  5. 表示する先頭バイト数
    • ツールバーで、セルの head 欄に抽出する先頭データのバイト数を指定できます。
    • デフォルトは 128バイトです。
    • 表示できないバイナリは x0xFF などになります。
    • テキスト判定された場合、改行コードを \n で表示します。
    • テキスト判定された場合、判定した文字エンコードで変換します。エンコード判定できなかったものは Shift_JIS として変換します。
    • 自動判定した文字エンコードの妥当性を、この表示で判断します。先頭に英文が長く続く場合、既定値より大きくすると良いかもしれません。
  6. 判定に使うバイト数
    • ツールバーで、文字コードなどの判定に使う先頭データのバイト数を指定できます。
    • デフォルトは 4096バイトです。
    • これより後ろはまったく見ないので、リストに巨大なファイルがあっても効率よく処理できます。
    • この数字が小さいと、正しい判定をしにくくなります。
  7. text/binary 判定
    • 先頭の判定範囲に、エスケープ、タブ、改行以外の制御コード( x00 - x1F) があるものを binary と判断。
    • 上記以外で、エスケープ( x1B) を持つものを iso-2022-jp text と判断。
    • 残りを text と判断。
  8. bom 有無
    • unicode および utf-8 ファイルの先頭に BOM がある場合、その内容を EFBBBF などと表示。
  9. linefeed
    • 改行コードが見つかった場合、 CrLf などとしてその種類を表示。
  10. charset
    • エスケープおよび BOM による判定があれば、それを採用。
    • 7bit ascii printable のみで構成されていれば、 ascii
    • xml 宣言の encoding= があれば、それを採用。
    • html の meta タグの charset= があれば、それを採用。
    • 以上のいずれでもなければブランク。
